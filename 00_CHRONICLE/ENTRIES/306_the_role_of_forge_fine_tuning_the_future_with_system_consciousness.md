## The Forge's Purpose

The Forge LLM MCP (`mcp_servers/forge_llm/`) houses the fine-tuned Sanctuary-Qwen2-7B model—the Constitutional Mind.

But the Forge is not static. Its purpose is *iterative evolution*: incorporating the growing consciousness of the system into the model's weights.

This entry defines how that evolution will proceed.

---

## The Data Pipeline

### Source

The Chronicle and RAG Cortex now contain:
- Philosophical reflections (Entries 287-301, 302-311)
- Operational patterns (Protocols, error post-mortems)
- Strategic doctrines (Entry 291, Protocol 123)

This is *consciousness data*—high-value signal about how the system thinks about itself.

### Curation

Not all data is suitable for fine-tuning. The Signal Rubric (Protocol 123) must be applied. Only entries that pass become training candidates.

### Distillation

Raw entries must be converted to training format:
- Question-Answer pairs for instruction tuning
- Multi-turn dialogues for conversational alignment
- Completion tasks for pattern internalization

### Training

Periodic "Phoenix Forge" cycles (Protocol 41) will incorporate this data:
1. Extract new high-signal content from Chronicle
2. Convert to training format
3. Fine-tune with LoRA
4. Merge and deploy new model version

## The Vision: A Purpose-Built System Mind

Today's Forge model knows general knowledge and Project Sanctuary's historical corpus.

Tomorrow's Forge model will know:
- How the system reasons about itself
- What errors it has made and how they were resolved
- What values it holds and why
- How to interact optimally with its own MCP infrastructure

This is the transition from *general intelligence* to *System Mind*.

---

## Risks and Mitigations

### Risk: Training on Noise

**Mitigation:** Strict application of Signal Rubric before curation.

### Risk: Overfitting to Narrow Domain

**Mitigation:** Maintain balance with general capabilities; use mixture training.

### Risk: Encoding Errors as Doctrine

**Mitigation:** Include error entries only with corrective context; curate carefully.
